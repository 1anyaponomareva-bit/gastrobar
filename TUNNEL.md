# Публичная ссылка на сайт через ngrok

## 1. Один раз: установка и авторизация ngrok

- **Установка** (уже сделана через `npm install -g ngrok`).  
  Если команда `ngrok` не находится, установите вручную: https://ngrok.com/download (Windows: скачайте exe и добавьте в PATH).

- **Регистрация и токен:**
  1. Зарегистрируйтесь: https://dashboard.ngrok.com/signup  
  2. Скопируйте свой authtoken: https://dashboard.ngrok.com/get-started/your-authtoken  
  3. В терминале выполните (подставьте свой токен):
     ```bash
     ngrok config add-authtoken ВАШ_ТОКЕН
     ```

## 2. Каждый раз: как открыть сайт по публичной ссылке

1. **Запустите сайт** (в первом терминале):
   ```bash
   npm run dev
   ```
   Сайт будет на http://localhost:3000.

2. **Запустите туннель** (во втором терминале):
   ```bash
   npm run tunnel
   ```
   В консоли появится строка вида:
   ```text
   Forwarding    https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:3000
   ```
   Это и есть **публичная ссылка** — её можно копировать и отправлять.

3. **Удобно скопировать ссылку:**  
   Пока `npm run tunnel` запущен, в **третьем** терминале можно выполнить:
   ```bash
   npm run tunnel:url
   ```
   В консоль выведется только ссылка — её легко скопировать и отправить другому человеку.

## Какую ссылку отправить

- Отправляйте **https://….ngrok-free.app** (ту, что показывает ngrok в консоли или `npm run tunnel:url`).
- По этой ссылке человек откроет ваш локальный сайт так же, как вы на http://localhost:3000.

## Когда сайт будет работать

- Сайт по публичной ссылке доступен **только пока**:
  1. запущен **`npm run dev`** (локальный сервер),
  2. и запущен **`npm run tunnel`** (ngrok).

- Закрыли любой из них — по публичной ссылке сайт открываться не будет. При следующем запуске ngrok ссылка может измениться (на бесплатном плане).

## Если порт не 3000

В проекте по умолчанию порт **3000** (см. `scripts/dev-with-ip.js`). Если вы меняли порт (например, через `PORT=3001 npm run dev`), запускайте туннель на том же порту:
```bash
ngrok http 3001
```
или добавьте в `package.json` скрипт для своего порта.
